# Логи Asterisk: настройка и ротация

## 1. Что пишется в логи

В системе Asterisk настройка ведения логов определяется в **logger.conf**. Используются два основных режима:

- **Базовый** (`logger.messages.conf`) — включает каналы:
  - `messages` — уровни `notice`, `warning`, `error`, `security` (SIP-запросы, ошибки, события безопасности).
- **Полный** (`logger.full.conf`) — активируется отдельно (например, через `asterisk-full-logs-on`), добавляет уровни:
  - `debug`, `verbose` (записываются в файл `full`).

### Где хранятся логи

- Директория логов задаётся в `asterisk.conf` параметром `astlogdir` => `/var/log/asterisk`
- Основные файлы:
  - `/var/log/asterisk/messages` — основной лог событий
  - `/var/log/asterisk/full` — полный лог (если включён)

## 2. Как и с какой периодичностью очищаются логи

Очистка и ротация логов Asterisk настроена через **logrotate**. Конфигурация находится в: `configs/logrotate/asterisk`

### Параметры ротации

| Параметр | Значение | Описание |
|----------|----------|----------|
| Периодичность | `daily` | Ротация выполняется раз в сутки (ночью, по расписанию logrotate) |
| Хранение | `rotate 7` | Хранится 7 ротированных файлов: `messages.1` … `messages.7`, затем старые удаляются |
| Сжатие | `compress` + `delaycompress` | Старые файлы сжимаются в `.gz`, последний текущий остаётся несжатым |
| Лимит по размеру | не задан | Ротация только по времени (daily), размер файла не учитывается |

### Особенности

- Если за день файл `messages` или `full` вырастает до нескольких гигабайт (например, из-за активных ботов или ошибок), он **не будет принудительно ротирован до следующего дня**
- Остальные логи системы (**journal**, **syslog**, **Docker**, **Postgres**) в данном репозитории не настраиваются — их ротация и очистка управляются системными сервисами или контейнерными образами
